## 如何测试
```bash
# 1. 在main分支上尝试提交
$ git add .
$ git commit -m "测试提交"

# 2. 在其他分支上测试正常提交
$ git checkout -b test-branch
$ git add .
$ git commit -m "测试提交"
```

## 2025-10-19 新增pre-commit钩子防止在main分支提交
### What
实现Git pre-commit钩子，防止在main分支上进行直接提交，确保开发工作流程的规范性。

1. 创建pre-commit钩子脚本 `.git/hooks/pre-commit`
2. 脚本检测当前分支是否为main分支
3. 如果在main分支上，显示错误信息并阻止提交
4. 在其他分支上允许正常提交

### Why
防止开发者在main分支上意外提交代码，确保所有开发工作都在功能分支上进行，符合Git工作流最佳实践。

### How
使用shell脚本实现pre-commit钩子，通过`git branch --show-current`获取当前分支名，与"main"进行比较，如果匹配则阻止提交并显示提示信息。
